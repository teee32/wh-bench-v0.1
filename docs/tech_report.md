## 摘要

本文介绍 **wh-bench-v0.1**，一个面向中文工业安全规程领域的小规模、可溯源、人工抽检的问答评测集。我们从公开发布的中国国务院条例与人大常委会立法语料（`twang2218/chinese-law-and-regulations`, 22552 篇）中筛选出 50 篇核心工业安全相关文件，使用 LLM 蒸馏为 Q&A 对，经规则过滤与人工抽检后产出最终评测集。我们公开抽检通过率、基线模型 char-F1 与 LLM-judge 评分，作为后续中文领域 LLM 工业安全合规能力评测的种子数据集。

## 1. 动机

当前中文 LLM 评测主要集中在通用知识（CMMLU/C-Eval）、法律（LawBench）、医学（MedBench）等方向，**工业安全规程**作为高合规要求的垂直领域评测资源稀缺。该领域具有：

1. **答案明确**：规程条款多为规定性表述（"必须"/"不得"/"应当"），便于客观评判
2. **错误代价高**：合规问答出错可能导致安全事故，故对模型事实性要求极高
3. **政策对齐**：武汉市《支持人工智能 OPC 创新发展若干措施》（2026/2）、湖北省同名文件（2026/5）均明确支持「高质量行业数据集」建设

本工作即面向上述空白，构建首个公开的、源可溯的中文工业安全规程问答评测集。

## 3. 构建流程

### 3.1 源筛选 (`scripts/01_load_sources.py`)

从源语料 22,552 篇法规中按以下两步筛选：

**步骤 1：标题关键词匹配**
- 白名单（命中其一保留）：`安全生产/安全规程/操作规程/受限空间/动火/危险化学品/...`等 30+ 关键词
- 黑名单（命中即排除）：`公告/通报/答复/...`等
- 文本长度过滤：1,000 ≤ chars ≤ 80,000

→ 命中 **852 篇**

**步骤 2：质量评分排序**
对每篇打分：
- 高价值关键词加 3 分（如"操作规程"/"受限空间"/"危险化学品"）
- 中价值关键词加 1.5 分（如"安全生产"/"应急管理"）
- 标题前 4 字含地名（地方法规）减 3 分
- 部委发文加 2-4 分（应急管理/市监总局/工信部最高 4 分）
- 长度适中加 1 分

→ 取前 **50 篇**，主要来自国务院条例（28 篇）、全国人大常委会（3 篇）、省级人大常委会（17 篇）

### 3.2 LLM 蒸馏 (`scripts/02_distill.py`)

**切段策略**：
- 优先按"第X条"边界切分
- 每段目标 1,500 字符（上限 2,200，下限 400）
- 短段与邻段合并，并扩展 section 范围标记（"第一条—第三条"）

**蒸馏 prompt**：
- 系统提示要求"答案必须可以直接从原文引用或紧密改写"
- 禁止外部知识推断
- 优先考察：必须步骤 / 禁止行为 / 数值阈值 / 责任主体 / 适用场景
- 输出 JSON：`{question, answer, source_quote, category, difficulty}`

**模型与参数**：
- 蒸馏模型：`gpt-5.4-mini` (via Foxnio OpenAI-compatible endpoint)
- 温度：0.3
- 每段最多 3 QA，每篇最多 6 段
- 并发：4 worker，含 3 次指数退避重试

### 3.3 自动过滤 (`scripts/03_filter.py`)

依次应用 5 级确定性过滤：

| 过滤 | 规则 |
|---|---|
| 非空 | question / answer / source_doc 非空 |
| 长度 | 5 ≤ \|Q\| ≤ 200，3 ≤ \|A\| ≤ 600 |
| 拒答 | answer 不含「无法回答」「未提及」等 |
| 源接地 | answer 字符与 source_text 重叠 ≥ 30% |
| 去重 | rapidfuzz 问题相似度 ≥ 90 视为重复 |

### 3.4 人工抽检 (`scripts/04_review_app.py`)

本地 Flask web 界面：
- 显示问题 / 答案 / 原文段落 / 元数据
- 评审员可选：✓ 通过 / ✗ 否决 / ⤼ 跳过
- 支持否决备注（用于错误模式分析）
- 状态自动持久化（`reports/review_state.json`），可中断恢复
- 完成后输出 `data/filtered/qa_v0.1_reviewed.jsonl` + 抽检明细 CSV

### 3.5 基线评测 (`scripts/05_baseline.py`)

**两种评分模式**：

1. **char-F1**：字符集 F1（precision-recall 调和平均）
   - 优点：免费、可重复、对中文友好
   - 缺点：对同义改写敏感

2. **LLM-judge**（可选 `--judge`）：
   - 使用更强模型（如 GPT-5.5）做语义裁判
   - 输出 1 / 0.5 / 0 三档
   - 更贴近人类判断，但成本更高

## 6. 局限性

1. **覆盖偏向**：源语料以"条例"为主，**部委发布的「技术规范」「操作规程」覆盖不足**。下一步引入应急管理部、市监总局公开标准目录
2. **单一蒸馏模型**：所有 QA 来自 `gpt-5.4-mini`，可能存在风格偏差。后续 v0.2 将引入多模型 ensemble + 跨验证
3. **规模有限**：v0.1 主打"小而严"，规模不适合训练，仅适合评测/sanity check
4. **类别不平衡**：危险化学品 / 矿山 / 民用爆炸物品类条款较多，工业其他细分（如电气、机械加工）覆盖少
5. **评分指标**：char-F1 对中文同义改写敏感，建议结合 LLM-judge 综合判断

## 7. 路线图

- **v0.2**（计划 2026-Q3）：引入应急管理部 / 市监总局 / 工信部公开标准，规模到 500 条
- **v0.3**：增加多模型 ensemble 蒸馏，类别平衡采样
- **v1.0**：1,000+ 条，含分类、抽取、合规判断三类任务
